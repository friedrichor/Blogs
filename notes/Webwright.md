## 1. 核心设计哲学：用代码代替点击

### 博客原文

> 系统不是逐一预测点击位置，而是让模型通过编写代码来启动浏览器会话并探索网站。
>
> 持久化的制品不是浏览器会话，而是本地工作区中的代码和日志。
>
> 浏览器没有持久状态。每次 Playwright 运行都必须创建一个全新的浏览器会话，从头导航，并通过代码重建状态。

### 代码实现

**无持久化浏览器状态的约束**写在 System Prompt 中，强制模型通过代码而非会话状态维护上下文：

[src/webwright/config/base.yaml:109](src/webwright/config/base.yaml#L109)

**持久化制品是代码和日志，而非浏览器会话**。环境模块将代码和截图持久化到磁盘工作区，而不是保留浏览器进程：

[src/webwright/environments/local_workspace.py:16-44](src/webwright/environments/local_workspace.py#L16-L44)（`LocalWorkspaceEnvironmentConfig`）

工作区目录通过 `output_dir` 配置项绑定，每一步的 bash 命令都在此目录下执行，产出的文件（`final_script.py`、截图、日志）构成持久化制品。

**Local Browser 模式**下，每一步 Python 代码都在一个新的 `async_playwright()` 上下文中运行，执行完即关闭：

[src/webwright/environments/local_browser.py:432-450](src/webwright/environments/local_browser.py#L432-L450)（`_run_python_code`）

这段代码将模型生成的 Python 片段包装成一个 `async def __agent_step__(page, context, browser, playwright, task)` 函数来执行，每次调用都是独立的浏览器会话。

## 3. Agent 交互循环

### 博客原文

> 1. 运行器将当前上下文发送给模型
> 2. 模型返回解析后的动作：思考块 + Shell 命令块
> 3. 环境在本地工作区执行命令
> 4. 返回观测结果（终端输出、日志、截图、错误信息）
> 5. 循环持续直到任务完成

## 5. "提前完成"问题与 Self-Reflection 门控

### 博客原文

> "提前完成"问题：Agent 必须先生成自我反思配置、在新文件夹中运行最终脚本并记录日志、通过自身判断，然后才能发出 `done: true` 标志。

### 代码实现

**门控逻辑**：当模型尝试设置 `done: true` 时，`execute_actions` 先检查 Self-Reflection 门控：

[src/webwright/agents/default.py:400-424](src/webwright/agents/default.py#L400-L424)（`execute_actions` 方法）

**门控验证逻辑**检查以下条件：

[src/webwright/agents/default.py:202-268](src/webwright/agents/default.py#L202-L268)（`_tool_gate_error` 方法）

1. `final_runs/` 目录是否存在
2. 最新 `final_runs/run_<id>/` 中是否有 `self_reflect_result.json`
3. 该文件中 `predicted_label` 是否为 `1`（成功）

若任何条件不满足，方法返回一段详细的错误文本，告知模型需要执行哪些步骤。模型收到此错误后会继续执行修复流程。

- **阶段2**（[第 419-436 行](src/webwright/tools/self_reflection.py#L419-L436)）：将所有 `Reasoning` 和 `final_script_log.txt` 注入最终 prompt，一次性得出 `Status: success` 或 `Status: failure`

**配置开关**：是否要求 Self-Reflection 通过才能完成：

## 6. 上下文爆炸问题与历史压缩

### 博客原文

> "上下文爆炸"问题：较长的编码轨迹会迅速超出上下文限制，因此每 20 步将历史记录压缩为一条摘要。

### 代码实现

**历史压缩触发条件**在主循环中：

[src/webwright/agents/default.py:374-380](src/webwright/agents/default.py#L374-L380)

**压缩间隔配置**（默认 20 步）：

[src/webwright/config/base.yaml:91](src/webwright/config/base.yaml#L91)

**压缩实现**（`_compact_history` 方法）：

[src/webwright/agents/default.py:303-339](src/webwright/agents/default.py#L303-L339)

1. 保留原始 system 消息
2. 用当前消息历史 + 压缩指令 prompt 向模型发起一次额外的 LLM 调用
3. 将所有非 system 消息替换为一条包含摘要的 user 消息
4. 摘要失败不会终止运行（`except Exception: return`）

**压缩指令 prompt**（告知模型需要摘要哪些内容）：

[src/webwright/agents/default.py:16-29](src/webwright/agents/default.py#L16-L29)（`DEFAULT_SUMMARY_USER_PROMPT`）

摘要 prompt 要求模型保留：原始任务目标、关键文件路径、已满足/未满足的 Critical Points、已发现的有效选择器、最新 final_runs 状态、以及下一步行动。

**Local Browser 模式下的 ARIA 快照裁剪**（另一种减少上下文的机制）：

[src/webwright/agents/default.py:275-301](src/webwright/agents/default.py#L275-L301)（`_prune_old_observation_aria_snapshots` 方法）

[src/webwright/config/local_browser.yaml](src/webwright/config/local_browser.yaml)（`keep_last_n_observations` 配置项）

只保留最近 N 条观测的 ARIA 快照，旧的替换为占位符 `"(ARIA snapshot pruned; see most recent observation)"`。

## 9. 可重用 CLI 工具输出

### 博客原文

> 系统支持为每个用户任务输出参数化的 CLI 工具，供后续重复使用。
>
> 脚本可以被重新运行、改造和共享。

### 代码实现

**crafted_cli.yaml** 配置启用 CLI 工具模式：

[src/webwright/config/crafted_cli.yaml](src/webwright/config/crafted_cli.yaml)

此配置覆盖 System Prompt 和 Instance Template，指导 Agent 生成带参数解析的可复用脚本，而非单次执行的一次性脚本。

**技能文档**描述了两种使用模式：

**脚本可重用性**的实现基础是工作区目录结构：

[src/webwright/config/base.yaml:337-340](src/webwright/config/base.yaml#L337-L340)（`instance_template` 中关于 `final_runs` 目录管理的规则）

## 10. 性能评测与成本数据

### 博客原文

> **Online-Mind2Web**（300 个任务，136 个网站）：
> - GPT-5.4：准确率 86.67%（AutoEval 类别的最优水平）
> - Claude Opus 4.7：准确率 84.7%
> - 困难任务上 Claude 表现更好（80.5% vs 76.6%）
>
> **Odysseys**（200 个多步骤任务）：
> - Webwright + GPT-5.4：60.1%（比此前最优水平提升 35.1 个百分点）
> - 基础 GPT-5.4（无 Webwright）：33.5%（相对提升 79.4%）
>
> **成本**：GPT-5.4 平均每任务花费 2.37 美元，平均 26.3 步。前 50 步达到 82% 准确率；后 50 步额外增加 3-4 个百分点。

## 11. 设计原则：少即是多

### 博客原文

> 少即是多——随着模型能力的提升，过度工程化的 Web Agent 框架反而变得越来越鸡肋，甚至成为束缚。
>
> 只有一个单独的 Agent 循环，没有任何多 Agent 编排。

### 代码实现

**单 Agent 循环**，无图引擎或多 Agent 框架：

[src/webwright/agents/default.py:363-381](src/webwright/agents/default.py#L363-L381)（`while True` 主循环）

整个循环仅 19 行代码，逻辑清晰：`step()` → `InterruptAgentFlow` 异常处理 → `save()` → 检查退出条件 → 可选历史压缩。

**极简架构**——核心模块总共约 1500 行：

| 文件 | 行数 | 职责 |
|------|------|------|
| [src/webwright/agents/default.py](src/webwright/agents/default.py) | 468 | Agent 主循环 |
| [src/webwright/models/base.py](src/webwright/models/base.py) | 588 | LLM 通信基类 |
| [src/webwright/environments/local_browser.py](src/webwright/environments/local_browser.py) | 568 | Playwright 浏览器环境 |
| [src/webwright/run/cli.py](src/webwright/run/cli.py) | 155 | CLI 入口 |

**可组合的 YAML 配置**而非硬编码逻辑：

[src/webwright/run/cli.py:41-43](src/webwright/run/cli.py#L41-L43)（配置叠加）

[src/webwright/utils/serialize.py](src/webwright/utils/serialize.py)（`recursive_merge` 函数）

通过 `-c base.yaml -c model_claude.yaml -c local_browser.yaml` 叠加即可切换模型或浏览器模式，无需修改任何代码。

**提供者无关性**——同一个 Agent 循环在 OpenAI、Anthropic、OpenRouter 下无差别运行：

[src/webwright/models/anthropic_model.py](src/webwright/models/anthropic_model.py)（Anthropic 适配器）

[src/webwright/models/openai_model.py](src/webwright/models/openai_model.py)（OpenAI 适配器）

[src/webwright/models/openrouter_model.py](src/webwright/models/openrouter_model.py)（OpenRouter 适配器）

所有适配器均继承自 `BaseModel`，只需重载 5-6 个方法。
